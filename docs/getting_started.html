Getting Started

New to this project? Start here. This page guides you through the learning path step-by-step.

Learning Path

Follow this order for best understanding:

Step 1: Understand the Problem

Read Fundamentals (5 min). You'll learn:

  • What is speech recognition and digit classification?
  • High-level overview of MFCC, K-means, VQ, HMM.
  • How all pieces fit together.
Step 2: Why These Parameters?

⭐ NEW & IMPORTANT: Read Parameters Explained (8 min). You'll understand:

  • Why DIM=39? (MFCC standard: 13 + 13 delta + 13 delta-delta)
  • Why N=5 states? (phonetic phases in digit utterance)
  • Why K=100 (kmeans) vs K=32 (final)? (codebook design tradeoff)
  • Why M=32 symbols? (balance quantization detail vs overfitting)
  • Why iterations=15? (empirical convergence for digit HMM)
  • Why EPS=1e-4? (K-means convergence threshold)

This is tailored to YOUR code — not generic HMM. See exact values from kmeans.c, vq_batch.c, hmm.c.

Step 3: Unfamiliar Terms?

Refer to Glossary (2 min). Defines every term used in code and docs.

Step 4: High-level Project Structure

Read Project Overview (3 min). Describes:

  • Files in the repo.
  • What each C file does.
  • Data flow (MFCC → K-means → VQ → HMM).
Step 5: Hands-on Commands

Follow Step-by-step Tutorial (10 min). You'll learn:

  • Prerequisites (C compiler, Python).
  • Exact commands to compile and run each program.
  • Windows/WSL notes.
Step 6: Code Deep Dive

Read source code line-by-line:

  • kmeans.c — clustering algorithm.
  • vq_batch.c — maps MFCC to symbols.
  • hmm.c — HMM training and testing (most complex).

Read headers too: kmeans.h, hmm.h.

Step 7: Math Details (Optional)

If you want to understand the math behind forward algorithm, Baum–Welch, etc., see Math Appendix. Now includes worked examples with your actual parameters (5 states, 32 symbols).

  • Detailed derivations.
  • Worked example of forward algorithm.
  • References to papers.
Step 8: Experiment

Once you understand the code:

  • Modify constants (N=# states, K=# clusters, M=# symbols).
  • Change the number of Baum–Welch iterations.
  • Experiment with different initialization in K-means.
  • See how accuracy changes.

Prerequisites

Software:
  • C compiler: gcc, clang, or MSVC. On Windows, WSL (Windows Subsystem for Linux) recommended for POSIX compatibility.
  • Python (optional): to serve docs locally via `python -m http.server`.
  • Git (optional): to clone/manage the repo.
Knowledge:
  • C basics: arrays, loops, functions, structs. NOT advanced C (no threading, no complex memory management).
  • Linear algebra (light): vectors, matrices, distance. Math appendix provides context.
  • Statistics (light): probability, expectation. Fundamentals page explains needed concepts.

Quick Command Cheat Sheet

From project root `d:/e2e speech/project/website/assignment 6/HMM_Assignment_5/HMM/`:

# Compile HMM program
gcc -o hmm.exe hmm.c -lm

# Run HMM (trains on train.seq, tests on dev.seq)
.\hmm.exe

# Compile K-means (optional, for learning)
gcc -o kmeans.exe main.c kmeans.c -lm
.\kmeans.exe

# Compile VQ batch (needs WSL/MinGW for POSIX)
gcc -o vq_batch vq_batch.c -lm
./vq_batch

# View docs locally
python -m http.server 8000
# then open http://localhost:8000/docs/index.html
